# UX Copy Guidelines

> Agent 및 개발자가 UI 문구를 일관되게 작성하기 위한 가이드라인

## 1. 용어 일관성 규칙

### 캠페인 관련

| 상황 | ❌ 피할 것 | ✅ 사용할 것 | 이유 |
|------|----------|------------|------|
| 제출 기한 | 마감 | **제출 마감** | "마감"은 중의적 (캠페인 종료, 모집 마감 등) |
| 캠페인 상태 | 끝남, 완료 | **종료됨** | 상태를 명확히 표현 |
| 캠페인 기간 | 기간 | **운영 기간** | 제출 기한과 구분 |
| 진행 중 | 활성, 열림 | **진행중** | 한글 표기 통일 |

### 액션 버튼

| 상황 | ❌ 피할 것 | ✅ 사용할 것 | 이유 |
|------|----------|------------|------|
| 임시 저장 | 저장하기 | **임시 저장** | 최종 제출과 구분 |
| 최종 제출 | 완료, 저장 | **제출하기** | 행동 명확화 |
| 취소 | 돌아가기 | **취소** | 일관성 |
| 삭제 | 지우기 | **삭제** | 강한 행동 명시 |

### 상태 표시

| 상황 | ❌ 피할 것 | ✅ 사용할 것 |
|------|----------|------------|
| 미제출 | 안함, 없음 | **미제출** |
| 제출 완료 | 완료, 끝 | **제출 완료** |
| 평가 대기 | 대기 | **평가 대기** |
| 평가 완료 | 끝, 완료 | **평가 완료** |

## 2. 아이콘 매핑 표준

### 시간/날짜 관련
- ⏳ 마감 임박, 시간 경과
- ⏰ 제출 마감, 기한
- 📅 날짜, 기간
- 🗓 캘린더, 일정

### 상태 관련
- ✅ 완료, 성공
- ⚠️ 주의, 경고
- ❌ 실패, 거부
- 🔄 진행중, 동기화
- ⏸️ 일시정지, 대기

### 액션 관련
- 📝 작성, 편집
- 💾 저장
- 🗑️ 삭제
- 📤 제출, 전송
- 📥 다운로드

### 사용자/팀 관련
- 👤 개인
- 👥 팀, 그룹
- 🏢 조직, 부서

## 3. 날짜/시간 표기 규칙

### 표준 포맷

```typescript
// 짧은 형식 (사이드바, 목록)
format(date, 'MM.dd HH:mm')  // "11.27 23:59"

// 중간 형식 (카드, 상세)
format(date, 'yyyy.MM.dd')   // "2025.11.27"

// 긴 형식 (모달, 상세 페이지)
format(date, 'yyyy년 M월 d일 HH:mm')  // "2025년 11월 27일 23:59"
```

### 상대 시간 표현

| 남은 시간 | 표시 | 스타일 |
|----------|------|--------|
| 24시간 이내 | `D-0` 또는 `오늘 마감` | 🔴 빨간색, bold |
| 3일 이내 | `D-N` | 🟠 주황색 |
| 7일 이내 | `D-N` | 🟡 노란색 |
| 7일 초과 | 날짜만 표시 | 기본 스타일 |

## 4. 에러/성공 메시지

### 성공 메시지

```
✅ 제출이 완료되었습니다.
✅ 임시 저장되었습니다.
✅ 삭제되었습니다.
```

### 에러 메시지

```
❌ 제출에 실패했습니다. 다시 시도해주세요.
❌ 필수 항목을 입력해주세요.
⚠️ 마감 기한이 지났습니다.
```

### 확인 메시지

```
정말 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다.
제출하시겠습니까? 제출 후에는 수정할 수 없습니다.
```

## 5. Agent 적용 규칙

### code-writer, error-fixer 등 구현 Agent

1. **새 UI 문구 작성 시**: 이 가이드라인의 용어 사용
2. **기존 문구 수정 시**: 일관성 검사 후 가이드라인 준수
3. **아이콘 선택 시**: 매핑 표준 참조

### 검증 체크리스트

- [ ] "마감"만 단독 사용하지 않았는가? → "제출 마감" 사용
- [ ] 액션 버튼이 명확한가? → "저장하기" 대신 "임시 저장" 또는 "제출하기"
- [ ] 아이콘이 의미와 일치하는가?
- [ ] 날짜 포맷이 일관적인가?

## 6. 실제 적용 사례

### 캠페인 사이드바 (2025-11-27 수정)

**Before:**
```tsx
<span>마감: {format(date, 'MM.dd HH:mm')}</span>
```

**After:**
```tsx
<span>제출 마감: {format(date, 'MM.dd HH:mm')}</span>
```

**파일:**
- `widgets/spark-note-sidebar/ui/CampaignItem.tsx:132`
- `features/admin/campaigns/ui/CampaignCard.tsx:114`
- `features/campaign-status-banner/ui/CampaignProgressBanner.tsx:115`
